## Known solver issues

**SSP2-4.5 stabilization — secant collapse at period 20 (2095).** `forward-look=2` + stabilization locks periods 18–19 at interpolated tax values, leaving period 20 to solve in isolation. The response function flattens → secant denominator collapses → carbon price diverges or hits NaN. This is GCAM GitHub Issue #578, open and unresolved in v8.2. **Workaround confirmed working:** add `<overshoot/>` and `<max-tax>8000</max-tax>` to the policy target file. The resulting run is NOT true stabilization (forcing peaks ~4.526 W/m² at 2090). For publication, label accordingly or attempt `<forward-look>0</forward-look>` as an alternative (untested as of 2026-03-26).

**BaseX lock file error after first DB write.** Symptom: `database_basexdb: lock file does not exist` at `DiskData.finishUpdate`. Blocks all subsequent writes in the same session. Fix: run a BaseX OPEN/OPTIMIZE/CLOSE cycle on `database_basexdb` to reset lock state before re-running.
